# 3차원 측정데이터 AI 분석 — 솔루션 제안서

> 대상: 품질팀 / 경영진 보고용
> 한 줄 요약: **CMM(3차원 측정기)에서 나오는 측정 리포트를 자동으로 엑셀로 정리하고, 추세·이상·공정능력을 분석해 "불량이 터지기 전에" 잡는 시스템.**

---

## 0. 지금 가진 데이터가 뭔가요? (한눈에 이해하기)

`3d` 파일은 **엔진 블록(THETA 2.5 / 2.4)을 3차원 측정기(CMM, 장비명 pc-dmis)로 잰 성적서**입니다.
사진2·3의 PDF와 똑같은 내용이고, PDF로 받아도 동일하게 처리됩니다.

측정기는 도면에 정해진 **기준치수(설계값)** 대비 **실제로 가공된 값**이 얼마나 차이 나는지를 항목마다 찍어줍니다. 핵심 용어 5개만 알면 됩니다.

| 리포트 용어 | 뜻 | 비유 |
|---|---|---|
| **NOMINAL(기준값)** | 도면이 요구하는 정답 치수 | 과녁의 정중앙 |
| **+TOL / -TOL(공차)** | 허용 오차 범위(±) | 과녁에서 "여기까진 OK" 원 |
| **MEAS(측정값)** | 실제 가공된 값 | 화살이 박힌 위치 |
| **DEV(편차)** | 측정값 − 기준값 (얼마나 빗나갔나) | 중앙에서 벗어난 거리 |
| **OUTTOL(공차초과)** | 공차를 넘어버린 양. **0보다 크면 불합격** | 과녁 밖으로 나간 정도 |

> 항목 종류도 위치(Location), 위치도(Position), 평면도(Flatness), 직각도(Perpendicularity),
> 진원도/원통도(Circularity/Cylindricity) 등으로 나뉘는데, 분석 관점에선 **전부 "DEV가 공차 안이냐"** 하나로 통일해서 봅니다.

**이번 `3d` 파일 1장 안에 들어있던 것:**
- 측정항목 **604개**
- 그중 **불합격(NG) 42개 (7.0%)**
- 합격이지만 공차의 70% 이상을 써버린 **"예비 불량" 33개**

## 1. 1단계 — "엑셀로 깔끔하게 정리" (요청하신 그것, 완료)

지금 `3d`/PDF는 사람이 눈으로 읽기 힘든 줄글입니다. 이걸 자동으로 **표 한 장**으로 바꿨습니다.

산출물: **`out/측정데이터_정리.xlsx`** — 시트 6개

| 시트 | 내용 |
|---|---|
| 안내 | 전체 요약(총 항목/불합격/위험 수) |
| 전체데이터 | 604개 전 항목, 필터·정렬 가능 |
| 요약 | 공정(섹션)별 합격·불합격 집계 |
| 불합격(NG) | 규격을 벗어난 42개만 |
| **위험관리** | 합격이지만 공차 70%+ 소진한 33개 → **다음에 터질 후보** |
| 추세 | 같은 부위를 여러 번 측정한 항목(추세 분석 대상) |

> 공차소진율 열은 **녹→황→적** 색으로 자동 표시됩니다. 100% = 규격 한계.
> PDF로 데이터가 와도 `python3 run.py 파일.pdf` 한 줄이면 같은 엑셀이 나옵니다.

## 2. 2단계 — "추세선·오르내림 보기" (요청하신 그것, 완료)

산출물: **`out/charts/00_대시보드.png`** (아래 4개를 한 장에)

1. **① 공정별 불합격 Pareto** — 어디부터 잡아야 효과가 큰지. 불합격의 대부분이 **`CYL BORE(실린더 보어)` 공정 한 곳에 몰려** 있습니다(16건). 여기만 잡아도 절반이 사라집니다.
2. **② 공차 소진율 분포** — 전 항목이 규격 한계(100%) 대비 어디쯤 있는지. 70~100% 노란 구간(예비 불량)이 보이면 "곧 넘어갈 항목"입니다.
3. **③ 공정별 편향** — 측정값이 **한쪽 방향으로 쏠려 있으면** 우연이 아니라 **공구 마모나 셋업 오프셋** 신호입니다. 한 공정이 통째로 "작게/크게" 깎이고 있으면 그 공구만 보정하면 됩니다.
4. **④ 실린더 보어 직경 추세** — 측정 회차(시간) 순으로 본 그래프. **첫 회차엔 보어 직경이 규격을 넘었다가(NG), 다음 회차엔 정상(OK)으로 내려온** 모습이 그대로 보입니다. → 추세선이 우하향하며 공차 안으로 복귀.

추가: **`05_SPC관리도.png`** — 같은 부위를 반복 측정한 데이터로 관리도(I-MR)를 그려, "관리 한계선(UCL/LCL)"을 벗어나는 순간을 잡습니다.

## 3. 핵심 발견 (이번 데이터에서 실제로 나온 것)

- **불합격의 약 40%가 실린더 보어(CYL BORE) 한 공정에 집중** → 개선 1순위.
- **첫 측정 묶음에서 보어 직경이 일제히 +방향 초과(NG) → 다음 묶음에서 정상화.** 전형적인 **공구 마모 → 교체/보정** 패턴. 이게 바로 "추세를 미리 봤으면 막을 수 있었던" 사례입니다.
- **REARMTGHOLE·LH BOSS 등 일부 공정은 편차가 한 방향으로 쏠림** → 개별 불량이 아니라 **셋업/공구 보정 한 번**으로 잡을 문제.
- 합격이지만 위험한 **예비 불량 33개**가 이미 존재 → 다음 로트에서 불합격으로 넘어갈 가능성.

## 4. 3단계 — "센세이셔널하게" (제가 제안하는 차별화 분석)

사진1 사장님 자료의 "AI 활용 방안"을 **실제로 굴러가는 기능**으로 옮긴 로드맵입니다.

### A. 실시간 SPC + 공정능력지수(Cpk/Ppk) — *측정과 동시에 합격/불합격을 넘어 "공정이 건강한가"*
- 항목별 관리도(X-bar/I-MR)로 **규격 초과 전에** 이상점·연속 상승 추세를 경보.
- **Cpk** 한 숫자로 "이 공정이 얼마나 여유 있게 합격하는가"를 정량화 → 고객/감사 대응 자료로 그대로 사용.

### B. 추세 기반 불량 예측 — *"이 추세면 N개 뒤에 규격 초과"*
- 각 항목 편차의 **기울기(추세)** 를 계산해, 공구 마모처럼 서서히 나빠지는 항목의 **초과 도달 시점을 예측**. → 공구를 "터지기 전에" 교체.

### C. 체계적 편향 → 원인 자동 연결 — *사진1의 "원인분석"*
- 한 방향 쏠림/동시에 같이 틀어지는 항목들을 **군집(clustering)** 으로 묶어 **동일 공구·픽스처·셋업** 으로 원인을 좁혀 제시.
- 설비ID/공구ID/작업조건 데이터를 같이 적재하면 "어느 호기·어느 공구에서 불량률이 높다"까지 자동 도출.

### D. 데이터 자동 축적 + 부품 추적성 — *사진1의 "데이터 통합관리"*
- 측정기 PDF가 나오면 **자동 적재 → 일/주/월 대시보드**. 부품 시리얼·일자·호기별 이력 조회.

### E. 자동 리포트 & 알림
- 불합격/예비불량 발생 시 **담당자 자동 통보(메일·메신저)**, 일일 품질 요약 자동 발송.

### F. 3D 시각화(중장기)
- 블록 형상 위에 편차를 **색(히트맵)** 으로 얹어 "어느 면이 휘었는지" 직관적으로 표시. → 사진1의 "실린더 보어 직경 증가 추세 분석 용이" 항목.

> **추천 우선순위:** 1단계(엑셀 자동화)·2단계(추세/대시보드)는 **이미 완성**.
> 다음은 **A(SPC/Cpk) → B(예측) → C(원인연결)** 순서가 투자 대비 효과가 큽니다.
> 단, A·B·C·E는 **데이터가 여러 장 쌓여야** 진짜 위력이 나옵니다(추세=시계열). → 지금부터 **모든 측정 PDF를 한 폴더에 모으는 것**이 1번 할 일.

## 5. 사용 방법 (그대로 따라하면 됩니다)

```bash
# 1) 준비물 설치 (최초 1회)
pip install -r requirements.txt

# 2) 분석 실행 — 기본 3d 파일
python3 run.py

# 3) 앞으로 PDF가 쌓이면 (여러 개 한꺼번에)
python3 run.py data/*.pdf

# 결과
#   out/측정데이터_정리.xlsx   ← 엑셀
#   out/charts/00_대시보드.png ← 추세/분석 그림
```

## 6. 폴더 구조

```
.
├─ 3d                       # 원본 측정 리포트(텍스트)
├─ run.py                   # 원클릭 실행
├─ requirements.txt
├─ src/
│   ├─ parse_cmm.py         # 리포트(텍스트/PDF) → 표 파서
│   ├─ build_excel.py       # 엑셀 생성
│   └─ analyze.py           # 추세·분석·차트
└─ out/                     # 결과물(엑셀 + 차트)
```
